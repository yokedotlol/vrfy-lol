# vrfy.lol — Gotchas

> Things that bite you if you forget them.

## 1. POST-only for validation

Email validation is POST-only (`POST /`). If someone tries `GET /user@domain.com`, they get a 405 with guidance to use POST. This is a privacy design — emails never appear in URLs, server logs, CDN analytics, or browser history.

## 2. Domain-level caching, not email-level

KV cache keys are **domain-only** (7-day TTL). Local parts are never persisted. This means all emails at the same domain share cached DNS/MX results. The `force` parameter bypasses the domain cache.

Extended validation (existence signals) uses a separate HMAC-keyed cache (30-day TTL) — the HMAC pseudonymizes the email so the raw address isn't stored, but results are per-email.

## 3. Microsoft GetCredentialType throttling

Microsoft's GetCredentialType endpoint can throttle at any time. The validator checks the `ThrottleStatus` field in the response — if throttled, it fails open (no existence signal, not an error). Don't retry on throttle; respect it.

## 4. EmailRep.io daily cap

Free tier: 250/month, 10/day. Platform key is self-capped at 8/day to leave headroom. The `EMAILREP_KEY` env var (BYO key) bypasses the platform cap. Smart cache TTL: high-rep emails cached 30 days, low/none cached 3 days.

## 5. Privacy relays are NOT disposable

Apple Hide My Email, SimpleLogin, Addy.io, Firefox Relay, DuckDuckGo — these set `privacy_relay: true` without setting `disposable: true`. They're legitimate relay services, not throwaway addresses. The `action` will be `allow` or `verify`, not `block`.

## 6. FakeFilter is API-only

FakeFilter has no license on their repo. The disposable domain list is from CC0/MIT sources only (141K+ domains). FakeFilter data is consumed via API at runtime only — never bundled.

## 7. Service bindings are optional

Both `EXTENDED_VALIDATION` and `YOKE` bindings are optional. The worker detects if they're available and falls back to inline checks (or `existence: null` for extended signals). Self-hosters can remove them from `wrangler.toml` — vrfy works standalone.

## 8. PoW challenges are stateless

PoW uses HMAC-SHA256 with a server-side secret, 5-minute time buckets, and IP binding. No nonce storage — the server can verify a solution without database state. The nonce-freshness check prevents replay within a bucket window.

## 9. SPA is server-rendered

The HTML is generated by `spa.ts` and served by the Worker directly — no separate client build, no Vite, no bundler. To change the HTML shell, edit `spa.ts`. Static pages (about, docs, privacy, terms, etc.) are all in the SPA_PATHS set.

## 10. Contact email is hello@yoke.lol

All .lol family tools use `hello@yoke.lol` as the contact email. vrfy.lol email routing was planned but never configured. Don't set up separate `hello@vrfy.lol` routing unless there's a specific need for a separate inbox.
